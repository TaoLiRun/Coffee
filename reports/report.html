Store Closure and Customer Displacement Analysis: Luckin Coffee Research Report

Research Question and Goal

Research Question

What is the causal effect of a forced interruption in purchasing on future consumer demand at Luckin Coffee? Specifically, we study whether temporary store closures that prevent consumers from making planned purchases — what we call displacement — cause a persistent reduction in post-closure purchasing through habit disruption (state dependence), or whether consumers fully return to their pre-closure purchase trajectories once the store reopens. We decompose the overall effect of closures into two channels:

  1. A displacement effect: the causal impact of the missed purchase itself on future behavior

  2. A baseline-demand effect: any shift in underlying demand unrelated to the missed purchase (e.g., discovering alternatives, or Luckin’s post-closure promotions)

Goal

To estimate the causal effect of store closures on customer behavior using:

  • A Difference-in-Differences (DiD) framework comparing affected vs. unaffected customers

  • A machine learning model to classify consumers into those who would have purchased during the closure in the counterfactual (displaced) versus those who would not (non-displaced)

  • A Triple Difference-in-Differences (DDD) specification to estimate displacement effects

Methodology Summary (Pseudo Code)

1. Store Closure Identification

# Identify store closures from transaction data
FOR each store:
    Find periods with no transactions >= 7 days
    Record closure_start, closure_end, closure_duration
FILTER closures:
    Keep closures with duration < 30 days
    Keep closures occurring after Aug 2020

# Build closure registry with matched controls
FOR each closure:
    Find treatment_store_id (closed store)
    Identify control_stores with:
        - Similar setup_time (opening date)
        - Never closed during analysis period
        - min_ctrl_treat_ratio >= 2.0
    Select top 5 control stores by setup time match

2. Customer-Store Preference Analysis

# Load transaction data
LOAD order_commodity_data (10.6M rows, 779K customers)
LOAD order_result_data (7.3M orders)

# Filter active customers
FILTER customers WITH >= 5 unique purchase days
    → 286,784 qualified customers

# Compute store preferences
FOR each customer:
    Count purchases at each store
    Identify preferred_store = most frequently visited store
    Compute preference_ratio = purchases_at_preferred / total_purchases

# Build treatment and control groups
FOR each closure:
    treatment_customers = customers with preferred_store == closed_store
    control_customers = customers with preferred_store IN control_stores

3. Trend visualization

# Build Event-Time Panel
FOR each closure:
    FOR each treatment and control customer:
        # Pre-closure periods: fixed 7-day periods (w = -4, -3, -2, -1)
        FOR each w in range(-window_weeks, 0):
            Calculate:
                - n_purchases = purchases / 7  # daily rate
                - group = treatment or control
                - t = w (week number relative to closure)

        # During closure: uses actual closure duration
        Calculate:
            - n_purchases = purchases / closure_duration  # daily rate
            - group = treatment or control
            - t = 0 (closure window)

        # Post-closure periods: fixed 7-day periods (w = 1, 2, 3, 4)
        FOR each w in range(1, window_weeks + 1):
            Calculate:
                - n_purchases = purchases / 7  # daily rate
                - group = treatment or control
                - t = w (week number relative to closure)

OUTPUTS:
    - closure_pair_registry.csv (22 closures with matched controls)
    - closure_trend_lines_w4.pdf (4-week window trends)
    - closure_trend_lines_w8.pdf (8-week window trends)

4. Displacement Classification Model

# Build training data
FOR each closure event:
    Define observation period:
        - 4 pre-periods, each of length = closure_duration_days
        - During period (actual closure duration)
    Label customers as:
        displaced = 1 if predicted to have made at least one Luckin purchase
                    during the closure window (counterfactual without closure)
        displaced = 0 otherwise
Note: label is unobservable for treated customers; model trained on pre-closure periods where label IS observed.

# Extract features:
    - purchase_frequency (pre-closure)
    - store_preference_ratio
    - closure_duration_days
    - recency of last purchase
    - demographic features (age, gender, member_type)
    - distance to alternative stores

# Train ML model
MODEL = XGBoostClassifier(binary:logistic, max_depth=6, eta=0.1, 500 rounds)
TRAIN model on training set (265K rows, 54K unique members)
GENERATE displacement_probability scores for all customers

5. Triple Difference-in-Differences (DDD) Estimation

# DDD Specifications
# Note: t=0 (closure window) is EXCLUDED from all regressions.
# All pre-closure and post-closure periods are included.
# Reference period is t = -1 in event-study specs.

# Fixed effects absorbed in all specs:
#   event_fe_id     : consumer-closure FE (phi_{ie})
#                     absorbs time-invariant terms including main effects
#                     of `treated`, `disp_binary`, `displacement_prob_centered`,
#                     and `closure_length_std` — do NOT include these as
#                     separate regressors
#   rel_t           : relative-time FE (omega_t)
#   calendar_month  : calendar-month FE (gamma_m)

# Pre-compute interaction columns (all vary within the consumer-closure panel):
post_X_treated           = post * treated
post_X_disp              = post * disp_binary
post_X_treated_X_disp    = post * treated * disp_binary
post_X_score             = post * displacement_prob_centered
post_X_treated_X_score   = post * treated * displacement_prob_centered

# Spec B — Collapsed binary DDD:
Y = delta_B * post_X_treated
  + beta    * post_X_disp
  + delta_D * post_X_treated_X_disp
  + event_FE + rel_t_FE + calendar_month_FE + epsilon

# Identifies:
#   delta_B  (post_X_treated):        baseline-demand effect
#                                     ATT for non-displaced treated consumers
#   delta_D  (post_X_treated_X_disp): displacement effect
#                                     additional post-closure change for
#                                     displaced treated vs non-displaced treated,
#                                     net of the same gap in control
#   beta     (post_X_disp):           natural level-shift for displaced consumers
#                                     (present in both treated and control;
#                                      reflects mean-reversion of frequent buyers)

# Spec C — Collapsed continuous-score DDD:
Y = delta_B * post_X_treated
  + beta_S  * post_X_score
  + delta_S * post_X_treated_X_score
  + event_FE + rel_t_FE + calendar_month_FE + epsilon

# Identifies:
#   delta_B  (post_X_treated):         baseline effect at mean propensity
#   delta_S  (post_X_treated_X_score): marginal displacement effect per unit
#                                      of centered propensity score
#   beta_S   (post_X_score):           propensity-purchase relationship,
#                                      pooled across treated and control

# Event-study specs (Specs A, B, C, D):
# Replace single post dummy with period-by-period dummies using
# i(rel_t, <var>, ref=-1) syntax:

# Spec A — Overall ATT:
Y = sum_{l != -1} delta_l * I(t=l) * treated
  + event_FE + rel_t_FE + calendar_month_FE + epsilon
# delta_l for l < -1: pre-trend test (should be jointly zero)
# delta_l for l > 0:  overall ATT combining both channels

# Spec B — Binary DDD event-study:
Y = sum_{l != -1} delta_B_l * I(t=l) * treated
  + sum_{l != -1} delta_D_l * I(t=l) * treated_X_disp
  + sum_{l != -1} beta_l   * I(t=l) * disp_binary
  + event_FE + rel_t_FE + calendar_month_FE + epsilon

# Spec D — Adds closure-length heterogeneity (augments Spec B):
# + sum_{l != -1} kappa_l * I(t=l) * treated_X_len
# + sum_{l != -1} theta_l * I(t=l) * treated_X_disp_X_len  (tXdXlen)
# + sum_{l != -1}         * I(t=l) * disp_X_len             (lower order)
# closure_length_std main effect absorbed by event_FE

Pre-trend Testing Methodology

The parallel trends assumption is fundamental to the DiD/DDD methodology, requiring that treatment and control groups would have followed similar trajectories absent the treatment. We test this assumption using a joint F-test approach.

For each event study specification, we estimate the event study regression:

\[Y_{ict} = \alpha + \sum_{t=-4}^{4} \gamma_t \cdot I(rel_t = t) \cdot treated_i + FE + \epsilon_{ict}\]

Then we test the null hypothesis that all pre-treatment coefficients are jointly zero:

\[H_0: \gamma_{-4} = \gamma_{-3} = \gamma_{-2} = 0\]

A high p-value (> 0.05) indicates no evidence against the parallel trends assumption.

Interpretation of Results: - p > 0.05: Cannot reject parallel trends assumption (✓) - p < 0.05: Potential violation of parallel trends (⚠)
