Store Closure and Customer Displacement Analysis: Luckin Coffee Research Report

Research Question and Goal

Research Question

What is the effect of store closures on customer purchase behavior in a coffee chain context? Specifically:

  1. Displacement Effect: When customers’ preferred stores close, do they switch to other stores in the network (displacement) or reduce their overall purchases?

  2. Heterogeneity: Which customers are more likely to displace versus reduce purchases?

  3. Long-term Impact: What are the lasting effects of store closures on customer purchase frequency and loyalty?

Goal

To estimate the causal effect of store closures on customer behavior using: - A Difference-in-Differences (DiD) framework comparing affected vs. unaffected customers - A machine learning model to predict customer displacement propensity - A Triple Difference-in-Differences (DDD) specification to estimate displacement effects - Event study analysis to examine dynamic effects over time

Methodology Summary (Pseudo Code)

1. Store Closure Identification

# Identify store closures from transaction data
FOR each store:
    Find periods with no transactions >= 7 days
    Record closure_start, closure_end, closure_duration
FILTER closures:
    Keep closures with duration < 30 days
    Keep closures occurring after Aug 2020

# Build closure registry with matched controls
FOR each closure:
    Find treatment_store_id (closed store)
    Identify control_stores with:
        - Similar setup_time (opening date)
        - Never closed during analysis period
        - min_ctrl_treat_ratio >= 2.0
    Select top 5 control stores by setup time match

2. Customer-Store Preference Analysis

# Load transaction data
LOAD order_commodity_data (10.6M rows, 779K customers)
LOAD order_result_data (7.3M orders)

# Filter active customers
FILTER customers WITH >= 5 unique purchase days
    → 286,784 qualified customers

# Compute store preferences
FOR each customer:
    Count purchases at each store
    Identify preferred_store = most frequently visited store
    Compute preference_ratio = purchases_at_preferred / total_purchases

# Build treatment and control groups
FOR each closure:
    treatment_customers = customers with preferred_store == closed_store
    control_customers = customers with preferred_store IN control_stores

3. Trend visualization

# Build Event-Time Panel
FOR each closure:
    FOR each treatment and control customer:
        # Pre-closure periods: fixed 7-day periods (w = -4, -3, -2, -1)
        FOR each w in range(-window_weeks, 0):
            Calculate:
                - n_purchases = purchases / 7  # daily rate
                - group = treatment or control
                - t = w (week number relative to closure)

        # During closure: uses actual closure duration
        Calculate:
            - n_purchases = purchases / closure_duration  # daily rate
            - group = treatment or control
            - t = 0 (closure window)

        # Post-closure periods: fixed 7-day periods (w = 1, 2, 3, 4)
        FOR each w in range(1, window_weeks + 1):
            Calculate:
                - n_purchases = purchases / 7  # daily rate
                - group = treatment or control
                - t = w (week number relative to closure)

OUTPUTS:
    - closure_pair_registry.csv (22 closures with matched controls)
    - closure_trend_lines_w4.pdf (4-week window trends)
    - closure_trend_lines_w8.pdf (8-week window trends)

4. Displacement Classification Model

# Build training data
FOR each closure event:
    Define observation period:
        - 4 pre-periods, each of length = closure_duration_days
        - During period (actual closure duration)
    Label customers as:
        - displaced = 1 if purchased from control_store during closure
        - displaced = 0 otherwise

# Extract features:
    - purchase_frequency (pre-closure)
    - store_preference_ratio
    - closure_duration_days
    - recency of last purchase
    - demographic features (age, gender, member_type)
    - distance to alternative stores

# Train ML model
MODEL = RandomForestClassifier()
TRAIN model on training set (265K rows, 54K unique members)
GENERATE displacement_probability scores for all customers

5. Triple Difference-in-Differences (DDD) Estimation

# Build estimation panel
FOR each closure:
    FOR each treatment and control customer:
        FOR each week in [-4, -3, -2, -1, 1, 2, 3, 4]:
            Calculate:
                - n_purchases (outcome)
                - treated (1 if treatment customer, 0 if control)
                - post (1 if week >= 0 relative to closure)
                - disp_binary (1 if displaced=1, 0 otherwise)
                - disp_score (displacement_probability)

# DDD Specifications
Binary model:
    Y = α + β1*treated + β2*post + β3*disp
        + β4*treated*post + β5*post*disp + β6*treated*post*disp
        + period_FE + event_FE + calendar_month_FE + ε

Score model:
    Y = α + β1*treated + β2*post + β3*score
        + β4*treated*post + β5*post*score + β6*treated*post*score
        + period_FE + event_FE + calendar_month_FE + ε

Event study:
    Y = α + Σ γ_t*I(rel_t=t)*treated
        + Σ δ_t*I(rel_t=t)*disp
        + Σ θ_t*I(rel_t=t)*treated*disp
        + period_FE + event_FE + calendar_month_FE + ε

Pre-trend Testing Methodology

The parallel trends assumption is fundamental to the DiD/DDD methodology, requiring that treatment and control groups would have followed similar trajectories absent the treatment. We test this assumption using a joint F-test approach.

For each event study specification, we estimate the event study regression:

\[Y_{ict} = \alpha + \sum_{t=-4}^{4} \gamma_t \cdot I(rel_t = t) \cdot treated_i + FE + \epsilon_{ict}\]

Then we test the null hypothesis that all pre-treatment coefficients are jointly zero:

\[H_0: \gamma_{-4} = \gamma_{-3} = \gamma_{-2} = 0\]

A high p-value (> 0.05) indicates no evidence against the parallel trends assumption.

Interpretation of Results: - p > 0.05: Cannot reject parallel trends assumption (✓) - p < 0.05: Potential violation of parallel trends (⚠)

Main Findings

  1. Displacement is the primary response: Most affected customers do not reduce overall purchases but switch to alternative stores in the network.

  2. Closure duration matters: Longer closures lead to higher displacement probabilities (33% → 49%).

  3. Convergence over time: After 4-8 weeks, treated customers’ purchase levels converge with control customers, indicating successful displacement.

  4. Triple difference is significant: The DDD interaction term (treated × post × displacement) is statistically significant (p<0.05), suggesting heterogeneous displacement effects across customers with different displacement propensity.

  5. Event study shows delayed effects: The most significant positive effect appears in week 3 after closure, suggesting an adjustment period.
